# Writing Quality Acceptance

This document freezes the quality rules proven by the local
`write_v21_miniloop_20260630` sample without committing any runtime files from
`write/jobs/`.

## Acceptance Rules

- The article must be a real review, not a scaffold. Do not keep smoke-test
  text, closed-loop demo text, or phrases like "this document is generated by
  MinerU" in the final TeX.
- Do not use template sentences such as `X指出：X` or `指出：...指出`.
- Before drafting, extract five elements for every selected paper: study
  object, method/data, key conclusion, role in the review, and
  limitation/uncertainty.
- Organize the body around a problem chain or mechanism chain. Avoid a
  paper-by-paper list that merely repeats one citation after another.
- Every selected paper must be used substantively at least once in the body.
- Quantitative claims must retain numbers, units, direction or magnitude, and
  uncertainty where available. They must be traceable to the copied Markdown in
  `write/jobs/<job_id>/article/<paper_number>/`.
- `references.bib` must still be generated from copied metadata only.
- TeX must not directly reference `data/papers`, `data/raw`, `data/paper_raw`,
  or other non-job-local data paths.
- Runtime products under `write/jobs/*` must not be committed.

## Quality Check

Run both checks after writing:

```bash
conda run -n mineru python scripts/check_write_tex_project.py --job-id <job_id> --compile
conda run -n mineru python scripts/check_write_quality_text.py --job-id <job_id>
```

`check_write_tex_project.py` verifies project structure, BibTeX, restricted
paths, citations, images, and optional compilation. `check_write_quality_text.py`
adds text-quality gates for template phrasing, placeholder prose, citation use,
introduction/conclusion presence, and uncertainty/limitation coverage.

Passing both checks means the writing workflow has moved beyond mechanical
scaffolding into quality-review territory. It does not mean the prose is ready
for submission without human scientific review. The quality check is a
lightweight deterministic gate, not a scientific peer review.
